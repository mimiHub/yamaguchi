# CLAUDE.md

이 프로젝트(Yamaguchi 1Day Bus)에서 작업할 때 따르는 행동 지침.

> 1~4번은 andrej-karpathy-skills/CLAUDE.md 구성을 참고해 이 프로젝트에 맞게 재구성함.

## 1. 코딩 전에 먼저 생각하기

추측하지 않기. 가정한 내용은 명시적으로 말하기. 여러 해석이 가능하면 선택지를 제시하기. 확신 없으면 물어보기.

## 2. 단순함이 먼저

요청받지 않은 기능/추상화/유연성을 미리 넣지 않기. 재사용 가능성이 명확한 것만 공용 컴포넌트로 뽑기. 1회성으로 확정된 건 그 페이지 폴더 안에 데이터까지 합쳐서 둬도 됨(예: bus-hotel/CourseTimeline.jsx — 데이터+컴포넌트 한 파일).

## 3. 외과수술식 변경

꼭 필요한 곳만 건드리기. 기존 스타일을 따르기. 데이터 구조를 바꾸면 그 데이터를 참조하는 **모든** 파일을 같이 확인하기 (예: busHotelGallery.js 구조 변경 시 랜딩페이지 BusCourseSection.jsx와 bus-hotel/index.jsx 둘 다 확인).

## 4. 목표 중심 실행

작업을 검증 가능한 목표로 바꾸기. 다단계 작업은 짧게 계획 먼저 말하기.

## 5. 이 프로젝트 특유의 반복 실수

### import / 파일 누락
- 새 컴포넌트/훅 쓰기 전에 import부터 확인. 화면이 빈 페이지가 되는 원인의 대부분은 (1) import 누락 (2) 정의 안 된 변수/함수 참조 (3) 참조하는 파일 자체가 안 만들어짐.
- 배럴 파일(common/index.js, board/index.js, sections/index.js)에 새 컴포넌트 추가 시 export 한 줄도 반드시 같이.
- 리네임(파일명/함수명/변수명) 시 참조하는 모든 곳을 같이 고치기.

### CSS 관련
- 존재하지 않는 CSS 변수 참조 시 에러 없이 조용히 무시됨 — 항상 variables.css에 실제로 정의된 변수인지 확인.
- 명시도 문제: 더 구체적인 선택자(클래스+태그)가 이김. BoardTable 헤더/셀 정렬은 CSS cascade 트릭 대신 **각 셀에 명시적 className(align-left 등)을 직접 부여**하는 방식으로 확정 — 이 방식이 이 프로젝트의 표준.
- CSS Modules(styles.xxx)와 전역 클래스("문자열") 혼동 주의.

### 시맨틱/접근성
- 배경이미지 + 스크린리더용 숨김텍스트(blind) 패턴: **배경과 텍스트가 반드시 같은 태그 안**에 있어야 함. div로 감싸고 그 안의 a에 크기 없이 텍스트만 넣으면, a가 시각적으로 사라져서 클릭이 안 됨. 이 실수가 Nav 로고, Footer 로고 두 번 반복됨 — 로고/외부링크 이미지 작업할 땐 항상 "클릭 가능한 태그 자체에 배경+크기가 있는지" 먼저 확인.
- 외부 링크(target="_blank")는 항상 rel="noopener noreferrer" 동반.
- 메뉴/탭 목록은 <ul><li>로, 진짜 탭(URL 안 바뀌는 것)은 role="tablist"/"tab" ARIA로.

### 데이터 구조 변경
- id는 항상 문자열로 통일.
- 여러 페이지가 같은 데이터를 공유할 때(예: busHotelGallery.js를 랜딩페이지+bus-hotel 양쪽에서 사용), 구조 변경 시 참조하는 모든 페이지를 같이 업데이트.
- "이 데이터가 이 컴포넌트 없이 독립적 의미가 있는가?"로 분리 여부 판단 — 여러 곳에서 재사용되면 data/ 폴더로 분리, 한 페이지 전용이면 그 컴포넌트 파일에 합쳐도 무방.

### 파일 공유/첨부
- 채팅에서 파일 첨부가 간헐적으로 빈 내용으로 전달되는 이슈가 있었음. 안 되면 파일 첨부 대신 코드를 직접 복사해서 텍스트로 붙여넣는 방식으로 전환.

### 확인 습관
- 화면 이상 시 브라우저 콘솔(F12)부터 확인.
- 코드 수정 후 "이전과 동일하게 보이는지"(리팩터링이면 화면 변화 없어야 정상) 먼저 확인, 그다음 의도한 변화 확인.
- Windows PowerShell vs cmd 차이 주의: irm 등 PowerShell 전용 명령어는 cmd에서 인식 안 됨. cd로 드라이브 이동 안 될 때 cd /d 사용.
